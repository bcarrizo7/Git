¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en archivos y coordinar el trabajo entre múltiples personas. Es como una máquina del tiempo para tu código, donde cada commit es un punto en el tiempo al que puedes volver. 

user@matrix:~$ git --version
git version 2.34.1

📚
Obtener Ayuda

Git ofrece herramientas integradas para obtener ayuda sobre sus comandos. Usa estos comandos para explorar la documentación oficial.

git help [comando]
Muestra la documentación detallada de un comando específico.
$ git help commit
git help --all
Lista todos los comandos disponibles en Git.
$ git help --all

⚙️
Configuración Inicial

Antes de usar Git, configura tu identidad y preferencias para personalizar tu experiencia.

git config --global user.name "Tu Nombre"
Configura tu nombre de usuario globalmente para los commits.
$ git config --global user.name "Alex Cyber"
git config --global user.email "tu@email.com"
Configura tu email para la identificación en plataformas como GitHub.
$ git config --global user.email "alex@cyber.dev"
git config --global init.defaultBranch main
Establece "main" como rama por defecto en lugar de "master".
git config --global core.editor [editor]
Configura tu editor preferido para mensajes de commit (e.g., VS Code, Vim).
$ git config --global core.editor "code --wait"
git config --list
Muestra toda la configuración actual de Git.
$ git config --list
user.name=Alex Cyber
user.email=alex@cyber.dev

Comandos Fundamentales

Estos comandos son la base para trabajar con Git y gestionar cambios en tu proyecto.

git init
Inicializa un nuevo repositorio Git en el directorio actual.
$ mkdir mi-proyecto && cd mi-proyecto
$ git init
Initialized empty Git repository in /home/user/mi-proyecto/.git/
git status
Muestra el estado del repositorio: archivos modificados, nuevos, o en staging.
$ git status
On branch main
Changes not staged for commit:
  modified: README.md
  new file: app.js
git add [archivo]
Añade archivos al área de preparación (staging) para el próximo commit.
$ git add README.md
$ git add .
git commit -m "Mensaje del commit"
Guarda los cambios preparados con un mensaje descriptivo.
$ git commit -m "Añade README inicial"
[main (root-commit) a1b2c3d] Añade README inicial
1 file changed, 10 insertions(+)
git commit -a -m "Mensaje"
Combina git add y git commit para archivos ya rastreados.
$ git commit -a -m "Actualiza README"

Flujo de Trabajo Básico

Inicializar

Crea un nuevo repositorio con git init.

Modificar

Edita archivos y verifica cambios con git status.

Preparar

Usa git add para preparar los archivos modificados.

Confirmar

Registra los cambios con git commit.

Mensajes de Commit Claros

Escribe mensajes descriptivos, e.g., "Añade validación de formularios" en lugar de "Cambios".

Usa .gitignore

Excluye archivos innecesarios como node_modules o .env.

Trabajando con Remotos

Conecta tu repositorio local con GitHub para colaborar y compartir tu código.

git remote add origin [url]
Conecta tu repositorio local a un repositorio remoto.
$ git remote add origin https://github.com/user/mi-proyecto.git
git push -u origin [rama]
Envía tus commits al repositorio remoto y configura el upstream.
$ git push -u origin main
Branch 'main' set up to track remote branch 'main' from 'origin'.
git pull
Descarga y fusiona cambios del repositorio remoto.
$ git pull origin main
git clone [url]
Clona un repositorio remoto a tu máquina local.
$ git clone https://github.com/user/mi-proyecto.git
Cloning into 'mi-proyecto'...
git remote -v
Lista los remotos configurados para tu repositorio.
$ git remote -v
origin https://github.com/user/mi-proyecto.git (fetch)
origin https://github.com/user/mi-proyecto.git (push)

🔀
Ramas (Branching)

Las ramas son esenciales para desarrollar características de forma aislada y colaborar en equipo.

git branch [nombre-rama]
Crea una nueva rama sin cambiar a ella.
$ git branch feature/nueva-funcion
git checkout [nombre-rama]
Cambia a la rama especificada.
$ git checkout feature/nueva-funcion
Switched to branch 'feature/nueva-funcion'
git checkout -b [nombre-rama]
Crea y cambia a una nueva rama en un solo paso.
$ git checkout -b feature/nueva-funcion
git branch -m [nuevo-nombre]
Renombra la rama actual.
$ git branch -m feature/nueva-funcion feature/mejorada
git branch -d [nombre-rama]
Elimina una rama local ya fusionada.
$ git branch -d feature/nueva-funcion
git branch
Lista todas las ramas locales.
$ git branch
* main
feature/nueva-funcion

Fusiones y Resolución de Conflictos

Las fusiones combinan ramas, pero los conflictos requieren intervención manual. Aprende a gestionarlos.

git merge [rama]
Fusiona la rama especificada en la rama actual.
$ git checkout main
$ git merge feature/nueva-funcion
git merge --abort
Cancela una fusión con conflictos.
$ git merge --abort
git diff
Muestra diferencias entre el estado actual y el último commit.
$ git diff
diff --git a/file.txt b/file.txt...
git merge --no-ff [rama]
Fuerza una fusión con un commit de merge, preservando el historial de la rama.
$ git merge --no-ff feature/nueva-funcion

Resolución de Conflictos

Edita los marcadores de conflicto (<<<<<<<, =======, >>>>>>>), luego usa git add y git commit.

Inspección y Comparación

Explora el historial y compara cambios para entender la evolución de tu proyecto.

git log
Muestra el historial completo de commits.
$ git log

git log --oneline --graph
Muestra el historial en una línea con un gráfico de ramas.
$ git log --oneline --graph

git show [commit]
Muestra detalles de un commit específico.
$ git show a1b2c3d
git diff [rama1] [rama2]
Compara diferencias entre dos ramas.
$ git diff main feature/nueva-funcion

🔙
Revertir Cambios

Aprende a deshacer errores sin perder el control de tu historial.

git reset [archivo]
Quita un archivo del área de staging, pero conserva los cambios.
$ git reset README.md
git reset --hard [commit]
Restablece el repositorio a un commit específico, descartando cambios posteriores.
$ git reset --hard a1b2c3d
git revert [commit]
Crea un nuevo commit que deshace los cambios de un commit específico.
$ git revert a1b2c3d

Cuidado con Reset

git reset --hard elimina cambios permanentemente. Usa git revert para deshacer de forma segura en ramas compartidas.

Referencia Rápida

Comando Descripción
git merge --no-ff Fusión con commit de merge
git log --oneline --graph Historial gráfico simplificado
git revert Deshace un commit

⚡
Rebase y Cherry-Pick

Herramientas avanzadas para reescribir el historial y seleccionar commits específicos.

git rebase [rama]
Reescribe el historial aplicando tus commits sobre otra rama.
$ git rebase main
Successfully rebased and updated refs/heads/feature/nueva-funcion
git rebase -i [commit]
Rebase interactivo para editar, reordenar o eliminar commits.
$ git rebase -i HEAD~3
git cherry-pick [commit]
Aplica un commit específico de otra rama.
$ git cherry-pick a1b2c3d
git rebase --onto [rama] [rama-base] [rama-objetivo]
Reubica una rama sobre otra base específica.
$ git rebase --onto main develop feature/nueva-funcion

Precaución con Rebase

Evita rebase en ramas compartidas, ya que reescribe el historial y puede causar conflictos.

Automatización con Hooks

Los Git Hooks automatizan tareas en eventos específicos, como commits o pushes. 

$ cd .git/hooks

$ mv pre-commit.sample pre-commit

$ echo '#!/bin/sh\nnpm run lint' > pre-commit
chmod +x .git/hooks/pre-commit
Da permisos de ejecución al hook pre-commit.
$ chmod +x .git/hooks/pre-commit

Hook Pre-Push

Ejecuta pruebas antes de git push para evitar errores en el remoto.

Hook Post-Commit

Notifica o ejecuta tareas tras cada commit.

Configura credenciales seguras para interactuar con repositorios remotos.

¿Cómo funciona Git por dentro? (Lo fundamental)

Git es un almacén de objetos dentro de la carpeta .git/. Todo se basa en snapshots inmutables identificados por un hash (SHA-1 o SHA-256). Hay tres áreas clave: Working Directory (tus archivos), Index/Staging (lo que vas a commitear) y HEAD (el último snapshot confirmado en la rama actual).

1) Objetos de Git

blob = contenido del archivo; tree = directorio (lista blobs/trees); commit = snapshot + metadatos (autor, fecha, padres, mensaje); tag = puntero con nombre (anotada o ligera).

.git/objects//
.git/refs/heads/   # ramas
.git/refs/tags/    # tags
HEAD               # referencia a la rama actual (e.g., refs/heads/main)

2) Referencias y ramas

Una rama es un puntero (ref) a un commit. Al hacer nuevos commits, la rama avanza apuntando al último commit. HEAD normalmente apunta a una rama; en detached HEAD, HEAD apunta directo a un commit.

# Ver a qué commit apunta HEAD
git rev-parse HEAD
# Ver dónde apuntan las refs
git show-ref --heads --tags

3) Staging (Index)

El index (o staging) es un archivo binario en .git/index que registra qué versiones exactas (hashes) se confirmarán en el próximo commit.

# Muestra diferencias entre WD, index y HEAD
git diff            # WD vs. index
git diff --staged   # index vs. HEAD

4) Packs y GC

Para eficiencia, Git comprime objetos y deltas en packfiles. git gc limpia y reempaca para ahorrar espacio y acelerar lecturas.

git count-objects -vH
git gc --aggressive  # con cuidado en repos grandes

5) Merge vs. Rebase (visión interna)

Merge: crea un commit con dos padres (historial ramificado). Rebase: reescribe los padres de tus commits para “recolgarlos” sobre otra base (historial lineal).

# Fast-forward: la rama sólo avanza el puntero
git merge --ff-only rama
# 3-way merge: crea commit de merge si hay divergencia
git merge rama

6) Remotos y refspec

Los remotos son colecciones de refs. La refspec define qué refs locales se sincronizan con cuáles remotas (e.g., refs/heads/*:refs/remotes/origin/*).

git ls-remote origin
git fetch origin 'refs/heads/*:refs/remotes/origin/*'

25 problemas comunes y soluciones rápidas

Un resuelve-rápido para emergencias de Git. Expande cada punto para ver el comando o procedimiento sugerido.

01. No tengo nombre/email configurado 
git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
02. Cometí con email equivocado (último commit) 
git commit --amend --author="Nombre "
03. Hice commit en la rama equivocada 
git switch la-rama-correcta
git cherry-pick 
git switch -
git reset --hard HEAD~   # si quieres borrar en la rama equivocada
04. Deshacer el último commit (conservar cambios) 
git reset --soft HEAD~
# o
git reset --mixed HEAD~   # mueve cambios al WD
05. Deshacer el último commit (descartar cambios) 
git reset --hard HEAD~
06. Recuperar archivo borrado 
git checkout HEAD -- ruta/archivo.ext
# o
git restore --source=HEAD -- ruta/archivo.ext
07. Guardar cambios sin commitear 
git stash push -m 'wip'
# luego
git stash pop
08. Resolver conflicto de merge 
Edita marcadores <<<<<<<, =======, >>>>>>>
Luego: git add . && git commit
09. Sincronizar con remoto antes de push 
git pull --rebase origin main
# evita merges innecesarios
10. Push rechazado; necesito forzar 
git push --force-with-lease
# más seguro que --force
11. Subí credenciales por error 
REVOCAR claves/tokens inmediatamente.
Reescribir historial (git filter-repo) y rotar secretos.
12. Añadí archivos pesados por error 
git rm --cached ruta/archivo
Eco en .gitignore y commit.
Si ya están en historia, usar filter-repo.
13. Eliminar archivo del historial 
git filter-repo --path ruta/archivo --invert-paths
# (instalar git-filter-repo)
14. Estoy en detached HEAD 
git switch -
# o crear rama desde ahí:
git switch -c fix/desde-detached
15. Upstream no configurado 
git push -u origin rama-actual
# o
git branch --set-upstream-to=origin/rama rama
16. Unrelated histories 
git pull --allow-unrelated-histories
17. fatal: not a git repository 
Ejecuta dentro de un repo o inicializa: git init
18. Permission denied (publickey) 
Configura SSH o usa HTTPS.
ssh-keygen -t ed25519
Sube la clave pública a GitHub.
19. PAT y autenticación 
Usa Token en lugar de contraseña.
Configura credential helper o gh auth login.
20. Remote tiene commits que yo no 
git pull --rebase origin rama
# resuelve conflictos y luego push
21. LF/CRLF advertencias 
git config core.autocrlf input     # macOS/Linux
# o
git config core.autocrlf true      # Windows
22. Cambiar rama por defecto a main 
git branch -m master main
git push -u origin main
# Ajusta en GitHub los defaults
23. Renombrar rama remota 
git branch -m vieja nueva
git push origin :vieja nueva
git push -u origin nueva
24. Hice clone shallow y ahora necesito todo 
git fetch --unshallow
25. Limpiar archivos no rastreados 
git clean -fd
# CUIDADO: elimina sin preguntar

Consejo de seguridad

Antes de ejecutar comandos destructivos (reset --hard, clean -fd), crea una rama de respaldo: git switch -c backup/antes-de-cambiar.
