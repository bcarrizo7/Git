1) Objetos de Git

2) Referencias y ramas

Una rama es un puntero (ref) a un commit. Al hacer nuevos commits, la rama avanza apuntando al √∫ltimo commit. HEAD normalmente apunta a una rama; en detached HEAD, HEAD apunta directo a un commit.

# Ver d√≥nde apuntan las refs

3) Staging (Index)

El index (o staging) es un archivo binario en .git/index que registra qu√© versiones exactas (hashes) se confirmar√°n en el pr√≥ximo commit.

# Muestra diferencias entre WD, index y HEAD

4) Packs y GC

Para eficiencia, Git comprime objetos y deltas en packfiles. git gc limpia y reempaca para ahorrar espacio y acelerar lecturas.

5) Merge vs. Rebase (visi√≥n interna)

Merge: crea un commit con dos padres (historial ramificado). Rebase: reescribe los padres de tus commits para ‚Äúrecolgarlos‚Äù sobre otra base (historial lineal).
