¿Qué es Git?

Git es un sistema de control de versiones distribuido que permite rastrear cambios en archivos y coordinar el trabajo entre múltiples personas. Es como tener una máquina del tiempo para tu código, donde cada commit es un punto en el tiempo al que puedes volver. 

user@matrix:~$ git --version
git version 2.34.1

⚙️
Configuración Inicial

Antes de empezar a usar Git, necesitas configurar tu identidad. Esta información se incluirá en cada commit que hagas.

git config --global user.name "Tu Nombre"
Configura tu nombre de usuario globalmente. Este nombre aparecerá en todos tus commits.
$ git config --global user.name "Alex Cyber"
git config --global user.email "tu@email.com"
Configura tu email. Es fundamental para la identificación en plataformas como GitHub.
$ git config --global user.email "alex@cyber.dev"
git config --global init.defaultBranch main
Establece "main" como rama por defecto en lugar de "master" (estándar moderno).
git config --list
Muestra toda la configuración actual de Git en tu sistema.

🚀
Comandos Fundamentales

git init
Inicializa un nuevo repositorio Git en el directorio actual. Crea la carpeta oculta .git donde se almacena toda la información del repositorio.
$ mkdir mi-proyecto && cd mi-proyecto
$ git init
Initialized empty Git repository in /home/user/mi-proyecto/.git/
git status
Tu comando más importante. Muestra el estado actual del repositorio: archivos modificados, nuevos, eliminados, y el estado del staging area.
$ git status
On branch main
Changes not staged for commit:
  modified: README.md
  new file: app.js
git add [archivo]
Añade archivos al área de preparación (staging) para ser incluidos en el próximo commit.
$ git add README.md
$ git add .
git commit -m "Mensaje del commit"
Guarda los cambios preparados en el repositorio con un mensaje descriptivo.
$ git commit -m "Añade README inicial"
[main (root-commit) a1b2c3d] Añade README inicial
1 file changed, 10 insertions(+)

Flujo de Trabajo Básico

Inicializar

Crea un nuevo repositorio con git init.

Añadir Cambios

Usa git add para preparar los archivos modificados.

Confirmar Cambios

Registra los cambios con git commit -m "mensaje".

🌐
Trabajando con Remotos

GitHub es una plataforma para alojar repositorios Git, permitiendo colaboración y control de versiones en equipo. Aquí aprenderás a conectar tu repositorio local con uno remoto.

git remote add origin [url]
Conecta tu repositorio local a un repositorio remoto en GitHub.
$ git remote add origin https://github.com/user/mi-proyecto.git
git push -u origin main
Envía tus commits locales al repositorio remoto, configurando la rama main como upstream.
$ git push -u origin main
Branch 'main' set up to track remote branch 'main' from 'origin'.
git pull
Descarga y fusiona los cambios del repositorio remoto en tu rama local.
$ git pull origin main
git clone [url]
Clona un repositorio remoto a tu máquina local, incluyendo todo su historial.
$ git clone https://github.com/user/mi-proyecto.git
Cloning into 'mi-proyecto'...

🔀
Ramas (Branching)

Las ramas permiten trabajar en diferentes versiones de un proyecto simultáneamente. Son esenciales para el desarrollo colaborativo y la experimentación.

git branch [nombre-rama]
Crea una nueva rama sin cambiar a ella.
$ git branch feature/nueva-funcion
git checkout [nombre-rama]
Cambia a la rama especificada.
$ git checkout feature/nueva-funcion
Switched to branch 'feature/nueva-funcion'
git checkout -b [nombre-rama]
Crea una nueva rama y cambia a ella en un solo paso.
$ git checkout -b feature/nueva-funcion
git branch -d [nombre-rama]
Elimina una rama local que ya ha sido fusionada.
$ git branch -d feature/nueva-funcion

Flujo de Trabajo con Ramas

Crear Rama

Usa git checkout -b feature/nueva para crear y cambiar a una rama.

Desarrollar

Realiza cambios, usa git add y git commit.

Push a Remoto

Envía la rama con git push -u origin feature/nueva.

Crear Pull Request

En GitHub, crea un Pull Request para fusionar tu rama.

🔄
Fusiones y Resolución de Conflictos

La fusión combina ramas, pero a veces surgen conflictos que debes resolver manualmente. Aprende a manejar estas situaciones con confianza.

git merge [rama]
Fusiona la rama especificada en la rama actual.
$ git checkout main
$ git merge feature/nueva-funcion
git merge --abort
Cancela una fusión en caso de conflictos que no deseas resolver.
$ git merge --abort
git diff
Muestra las diferencias entre el estado actual y el último commit, útil para resolver conflictos.
$ git diff
diff --git a/file.txt b/file.txt...

Resolución de Conflictos

Los conflictos ocurren cuando Git no puede fusionar automáticamente. Edita los archivos marcados con <<<<<<< y >>>>>>>, elige las versiones correctas, luego usa git add y git commit.

🔍
Inspección del Historial

Explora el historial de commits para entender los cambios en tu proyecto y revertir errores si es necesario.

git log
Muestra el historial completo de commits.
$ git log
commit a1b2c3d...
git log --oneline
Muestra el historial en una sola línea por commit.
$ git log --oneline
a1b2c3d Añade README inicial
git show [commit]
Muestra los detalles de un commit específico.
$ git show a1b2c3d

Commits Atómicos

Haz commits pequeños y enfocados. Esto facilita la revisión y la reversión de cambios.

Usa .gitignore

Crea un archivo .gitignore para excluir archivos innecesarios, como node_modules o .env.

⚡
Rebase y Cherry-Pick

Herramientas avanzadas como rebase y cherry-pick te permiten reescribir el historial de commits para un control más preciso.

git rebase [rama]
Reescribe el historial aplicando tus commits sobre otra rama.
$ git rebase main
Successfully rebased and updated refs/heads/feature/nueva-funcion
git cherry-pick [commit]
Aplica un commit específico de otra rama a la rama actual.
$ git cherry-pick a1b2c3d
git rebase -i [commit]
Rebase interactivo para editar, reordenar o eliminar commits.
$ git rebase -i HEAD~3

Precaución con Rebase

Evita usar rebase en ramas compartidas, ya que reescribe el historial y puede causar conflictos con otros colaboradores.

🛠️
Automatización con Hooks

Los Git Hooks son scripts que se ejecutan automáticamente en ciertos eventos de Git, como antes o después de un commit o push. 

$ cd .git/hooks

$ mv pre-commit.sample pre-commit

$ echo '#!/bin/sh\nnpm run lint' > pre-commit

Hook Pre-Commit

Ejecuta pruebas o linters antes de cada commit para garantizar la calidad del código.

Hook Post-Merge

Automatiza tareas después de una fusión, como actualizar dependencias.

Flujo de Trabajo Experto

Rebase Interactivo

Limpia tu historial con git rebase -i.

Cherry-Pick

Selecciona commits específicos con git cherry-pick.

Automatización

Configura hooks para tareas automáticas.

Push Seguro

Verifica todo antes de git push.
